<objective>
Swap the PAT in release.yml's "Create GitHub Release" step from `GITHUB_TOKEN` to `RELEASE_TOKEN` so that the `release: published` event actually fires for downstream workflows. This is the unblocker for Phase 21+ — without it, `publish-packages.yml` (created in plan 21-02) silently never runs.

Purpose: GitHub intentionally suppresses workflow events that are triggered by `GITHUB_TOKEN`-initiated actions to prevent infinite loops. Releases created with `GITHUB_TOKEN` therefore never emit a `release: published` event visible to other workflows. A user-classed PAT (`RELEASE_TOKEN`) bypasses this suppression. This is a one-line edit — but it is the keystone of the entire phase.

Output: `.github/workflows/release.yml` line 446 changed from `${{ secrets.GITHUB_TOKEN }}` to `${{ secrets.RELEASE_TOKEN }}`. No other lines touched.
</objective>
