<!-- Critical Nix-specific facts. -->

flake.nix LIVES AT THE REPO ROOT — NOT under `packages/`. The path is exactly `flake.nix`.
This is required because:
- `nix run github:sandwichfarm/nsyte` looks for `flake.nix` at the repo root by default
- Phase 26 CI commits hash updates to flake.nix in-place (NIX-05: no external repo push)

Linux dynamic linking (per RESEARCH.md "Critical fact" lines 354-357 and Pitfall 4):
- `deno compile` on Linux produces a DYNAMICALLY LINKED ELF binary
- The interpreter path is hard-coded to `/lib64/ld-linux-x86-64.so.2` (or aarch64 equivalent)
- On NixOS that path does not exist — autoPatchelfHook rewrites it to a Nix-store glibc
- glibc is the only buildInput needed (deno bundles its own runtime; no libstdc++/openssl deps)

Darwin (Mach-O):
- macOS binaries use Mach-O, not ELF
- They run unmodified on macOS (the dynamic loader handles Mach-O via `@rpath` and the system dyld)
- autoPatchelfHook MUST NOT be applied to Darwin (it would fail or no-op)

dontUnpack / dontBuild (per Pitfall 4):
- Release assets are bare executables (no .tar.gz extension)
- Without dontUnpack=true, Nix's default unpack phase tries to extract a non-archive and fails
- dontBuild=true is also required because there is no source to compile

SRI hash format (per RESEARCH.md "SRI hash format" line 361):
- Nix flakes use SRI: `"sha256-<base64>="` not `"<hex>"`
- Placeholder: `"sha256-PLACEHOLDER_HASH_X86_64_LINUX="` (and per-system variants below)
- The `sha256-` prefix and `=` suffix MUST be retained in the placeholder so the structural shape is identifiable for sed-substitution

System-to-asset mapping (release.yml lines ~82, 152, 227 — confirmed):
- x86_64-linux   -> nsyte-linux-${version}             (line ~82-83)
- aarch64-linux  -> nsyte-linux-aarch64-${version}     [NOT YET BUILT — placeholder + comment, Phase 26 must add release step]
- x86_64-darwin  -> nsyte-macos-x64-${version}         (line ~152-153)
- aarch64-darwin -> nsyte-macos-arm64-${version}       (line ~227-228)

Placeholder strings used in this file (each unique to its system):
- PLACEHOLDER_VERSION                          -- single literal occurrence at top of `let` block
- sha256-PLACEHOLDER_HASH_X86_64_LINUX=
- sha256-PLACEHOLDER_HASH_AARCH64_LINUX=
- sha256-PLACEHOLDER_HASH_X86_64_DARWIN=
- sha256-PLACEHOLDER_HASH_AARCH64_DARWIN=

stdenvNoCC vs stdenv:
- Use `pkgs.stdenvNoCC.mkDerivation` because there is no compilation step (no C/C++ toolchain needed)
- This matches the canonical bun pattern from nixpkgs (RESEARCH.md line ~349-350)
- `pkgs.stdenvNoCC.isLinux` is the platform predicate (NOT `pkgs.stdenv.isLinux` — match the same predicate to the same stdenv variant being used for mkDerivation)

Key structural facts (all per NIX-01 and locked decisions from CONTEXT.md and RESEARCH.md):

1. The flake declares `description`, `inputs.nixpkgs.url`, and `outputs`.
2. The `outputs` attribute uses the `forAllSystems` idiom over the four supported systems: x86_64-linux, aarch64-linux, x86_64-darwin, aarch64-darwin.
3. A `version` binding holds the literal placeholder string `PLACEHOLDER_VERSION`. Phase 26 CI will sed-substitute the real bare semver here. This single binding is interpolated into all four URLs.
4. A `sources` attribute set maps each system name to `{ url; hash; }`. The url uses `${version}` for the bare semver and `v${version}` in the path for the GitHub tag (matching the same convention as the AUR/Homebrew/Scoop templates from Plans 01 and 02).
5. The aarch64-linux entry includes the comment `# NOTE: aarch64-linux binary requires release.yml aarch64 build step (not yet present).` immediately above the entry — this signals to Phase 26 that it depends on Phase 22 adding the build step OR on building it themselves.
6. Each per-system derivation uses `pkgs.stdenvNoCC.mkDerivation` (not stdenv) because there is no compilation.
7. `nativeBuildInputs` and `buildInputs` are conditional on `isLinux = pkgs.stdenvNoCC.isLinux`. On Linux only: `pkgs.autoPatchelfHook` in nativeBuildInputs and `pkgs.glibc` in buildInputs. On Darwin: both lists are empty (via `pkgs.lib.optionals`).
8. `dontUnpack = true;` and `dontBuild = true;` are both set (release assets are not archives).
9. `installPhase` uses `install -m755 -D $src $out/bin/nsyte` — `-D` creates parent dirs, `-m755` sets executable mode, `$src` references the fetchurl result.
10. The `meta` attribute set declares description, homepage, license (MIT), platforms (= supportedSystems), and mainProgram = "nsyte".
11. Each system also exposes `default = self.packages.${system}.nsyte;` so `nix run .` works without an explicit attribute path.

Exact file content to write (use exactly this text — Nix is whitespace-tolerant but indent style must remain consistent for review):

```nix
{
  description = "nsyte — publish your site to nostr and blossom servers";

  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixpkgs-unstable";

  outputs = { self, nixpkgs }:
    let
      version = "PLACEHOLDER_VERSION";

      # Per-system source URLs and SRI hashes.
      # Phase 26 CI patches these placeholders on release via sed.
      sources = {
        "x86_64-linux" = {
          url = "https://github.com/sandwichfarm/nsyte/releases/download/v${version}/nsyte-linux-${version}";
          hash = "sha256-PLACEHOLDER_HASH_X86_64_LINUX=";
        };
        # NOTE: aarch64-linux binary requires release.yml aarch64 build step (not yet present).
        "aarch64-linux" = {
          url = "https://github.com/sandwichfarm/nsyte/releases/download/v${version}/nsyte-linux-aarch64-${version}";
          hash = "sha256-PLACEHOLDER_HASH_AARCH64_LINUX=";
        };
        "x86_64-darwin" = {
          url = "https://github.com/sandwichfarm/nsyte/releases/download/v${version}/nsyte-macos-x64-${version}";
          hash = "sha256-PLACEHOLDER_HASH_X86_64_DARWIN=";
        };
        "aarch64-darwin" = {
          url = "https://github.com/sandwichfarm/nsyte/releases/download/v${version}/nsyte-macos-arm64-${version}";
          hash = "sha256-PLACEHOLDER_HASH_AARCH64_DARWIN=";
        };
      };

      supportedSystems = builtins.attrNames sources;
      forAllSystems = nixpkgs.lib.genAttrs supportedSystems;
    in
    {
      packages = forAllSystems (system:
        let
          pkgs = nixpkgs.legacyPackages.${system};
          src = sources.${system};
          isLinux = pkgs.stdenvNoCC.isLinux;
        in
        {
          nsyte = pkgs.stdenvNoCC.mkDerivation {
            pname = "nsyte";
            inherit version;

            src = pkgs.fetchurl {
              inherit (src) url hash;
            };

            # autoPatchelfHook patches the ELF interpreter path for NixOS.
            # Only needed on Linux — Darwin uses Mach-O, no patching needed.
            nativeBuildInputs = pkgs.lib.optionals isLinux [
              pkgs.autoPatchelfHook
            ];

            # glibc is the only runtime dependency for a deno-compile binary.
            buildInputs = pkgs.lib.optionals isLinux [
              pkgs.glibc
            ];

            dontUnpack = true;
            dontBuild = true;

            installPhase = ''
              install -m755 -D $src $out/bin/nsyte
            '';

            meta = with pkgs.lib; {
              description = "Publish your site to nostr and blossom servers";
              homepage = "https://github.com/sandwichfarm/nsyte";
              license = licenses.mit;
              platforms = supportedSystems;
              mainProgram = "nsyte";
            };
          };

          default = self.packages.${system}.nsyte;
        });
    };
}
```
