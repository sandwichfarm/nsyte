## Cross-Phase Integration

Static integration is wired:

- `release.yml` uses `RELEASE_TOKEN` for release creation.
- `publish-packages.yml` has independent manager jobs for AUR, Homebrew, Scoop, Winget, and Nix.
- `packages/` is the source of truth for manager templates.
- Preserved Chocolatey, Debian, Flatpak, and Snap templates are intentionally not live-published in v1.6.

Runtime integration still needs a real release run and external package-manager
verification before the milestone should be completed.
